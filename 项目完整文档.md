### 1.1 ⚠️ 关键坑:mihomo 全局代理
主机跑着 **mihomo(clash.meta)TUN 模式全局代理**,会劫持所有出站流量:
- 配置目录 `~/litellm/clash/`,主配置 `config.yaml`,订阅 `sub.yaml`(`api.eeox.net`)。
- systemd 用户服务 `mihomo.service`。API:`http://127.0.0.1:9092`,密钥 `mihomo2026`。混合代理端口 7890,DNS 劫持 :53,TUN 接口 `Meta`(fake-ip 198.18.x)。
- 默认规则 `MATCH,PROXY` → 所有流量走海外订阅。
- **已加的 DIRECT 直连例外**(`rules:` 段):
  - `PROCESS-NAME,cloudflared,DIRECT`(原有)、`PROCESS-NAME,frpc,DIRECT`
  - `DOMAIN-SUFFIX` 直连:`natfrp.com / natfrp.io / natfrp.org / frp-top.com`、`papermc.io / arcadiatech.org / serverjars.com`
  - `IP-CIDR,103.91.208.133/32,DIRECT`(SakuraFrp 节点)
- **改规则后热重载**(不重启、不掉线):
  ```bash
  curl -X PUT "http://127.0.0.1:9092/configs?force=true" \
    -H "Authorization: Bearer mihomo2026" \
    -d '{"path":"/home/zuens2020/litellm/clash/config.yaml"}'
  ```
- **此代理引发的连锁问题**(后文详述):frpc 连国内节点被 EOF、Docker 容器 DNS 失效、Crafty 联网检查失败。**切勿全局关闭它**(Cloudflare 等其它服务依赖)。

## 3. 内网穿透(SakuraFrp 樱花穿透)

- **为何用它**:主机在 NAT 内网无公网 IP;Cloudflare 免费版**不能代理裸 TCP**(MC Java 是 TCP),只能代理 HTTP;playit.gg 国内延迟高 → 选**樱花穿透**(国内节点低延迟)。
- **客户端**:`~/sakurafrp/frpc`(版本 0.51.0-sakura-12.3,从官方 CDN `nya.globalslb.net` 下)。
- **systemd 用户服务**:`~/.config/systemd/user/frpc.service`,`ExecStart` 用 `frpc --natfrp_tls -n -f <访问密钥>:27900022`。
- **隧道**:名称 `NEC_mc`(ID 27900022)· TCP · 远程端口 18650 · 节点「枣庄多线PLUS-扩容2」(IP `103.91.208.133`)· 限速 10 Mbit/s。
- **连接地址**:**`frp-top.com:18650`**。
- **延迟**:服务器↔节点往返约 180ms(该机器离枣庄节点较远);生存够用。
- ⚠️ 坑:一开始 frpc 登录节点报 `EOF`,是 mihomo 把到国内节点的流量绕去海外。解法 = 上面 mihomo 的 `PROCESS-NAME,frpc,DIRECT` + 节点 DIRECT 规则 + `--natfrp_tls`。
- 常用:`systemctl --user status/restart frpc`、`journalctl --user -u frpc -f`。

## 4. Crafty 管理面板

- **部署**:Docker 容器 `crafty`(镜像 `registry.gitlab.com/crafty-controller/crafty-4:latest`),**`--network host`**,`restart unless-stopped`,数据卷 `~/crafty/{backups,logs,servers,config,import}`。
- **端口**:面板 `https://<host>:8443`(自签证书)、游戏 25565、8123(dynmap 预留)。
- **登录**:默认管理员凭据在 `~/crafty/config/default-creds.txt`(用户名 admin)。监控面板靠它调 Crafty API。
- **对外**:Cloudflare 公开主机名 `craft.zuens2020.work` → `HTTPS localhost:8443`,需勾 **No TLS Verify**(自签证书)解决 502。

### 4.1 ⚠️ 两个关键坑 + 持久化补丁
1. **容器 DNS 失效**:mihomo TUN 下,Docker 桥接容器的 DNS 整条不通(UDP/TCP :53 都被吞)→ Crafty 判定「无网络」、新建服务器转圈。**解法:改 `--network host`**,容器直接用宿主机正常的 systemd-resolved。
2. **Crafty 联网检查写死**:`check_internet()` 只 ping `ntp.org`/`google.com`(1s 超时),国内必失败。**解法:入口包一层 wrapper 每次启动给当前镜像打补丁**,把它改成查 `api.papermc.io`/`baidu.com`(5s)。
   - wrapper:`~/crafty/patches/entrypoint-wrapper.sh`,容器以它为 `--entrypoint`(Cmd `-d -i`),幂等、**删容器重建/更新镜像都不丢**。
3. **`auto_start: true`**(已通过 API 设)→ 开机/重启后 Crafty 起来约 10s 自动拉起游戏服。

### 4.2 Crafty API 速记
```bash
# 登录拿 token
PW=$(python3 -c "import json;print(json.load(open('/home/zuens2020/crafty/config/default-creds.txt'))['password'])")
TOKEN=$(curl -sk -X POST https://127.0.0.1:8443/api/v2/auth/login \
  -H "Content-Type: application/json" -d "{\"username\":\"admin\",\"password\":\"$PW\"}" \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['data']['token'])")
SID=4b6cb785-2d6f-452e-a26c-e4dae005b564
# 重启游戏服 / 备份
curl -sk -X POST https://127.0.0.1:8443/api/v2/servers/$SID/action/restart_server -H "Authorization: Bearer $TOKEN"
curl -sk -X POST https://127.0.0.1:8443/api/v2/servers/$SID/action/backup_server  -H "Authorization: Bearer $TOKEN"
```
- 注:**备份保留份数(Max Backups)只能在 Crafty 网页里改**(4.10.7 的 API 该接口 handler-not-found)。备份动作可用 API。

## 6. 自研监控面板(本项目核心代码)

- **文件**:`~/dashboard/dashboard.py`(纯 Python 标准库,无第三方依赖)。
- **服务**:systemd 用户服务 `dashboard.service`,端口 **8765**,Cloudflare → `monitor.zuens2020.work`(HTTP)。
- **代码仓库**:私有 `ZUENS2020/mc-server-monitor`,本地克隆 `~/Documents/mc-server/`。
- **定位演进**:早期聚合监控本机所有 Docker/systemd 服务;**现已精简为只监控 MC 服务器 + frpc 隧道**(`SERVICES` 字典 = `mc` + `frpc`)。
- **视觉**:1080P 常驻屏设计,工业风、**全直角**(`border-radius:0!important`)。

### 6.1 首页(MC 聚焦)
1. **Minecraft 服务器面板**(12 个等宽框,6 列×2):状态 / 在线人数 / TPS / MSPT / 已占用内存(RSS÷Xmx)/ 运行时长 / 难度 / 视距 / 验证(正版)/ 版本 / 隧道 / 连接地址。右上角「详情 / 日志 ›」进详情页。
2. **整机资源**:CPU / 内存 / 磁盘 / Swap / 网络速率 / 系统负载(带进度条)。
3. **性能趋势图**:近 60 分钟 CPU% / 内存% / 负载,纯 SVG。
4. **在线玩家墙**:每人一张卡(头像 minotar.net + 血量/饱食条 + 维度·等级·坐标 + 4 格护甲色块),列数随人数自适应,左侧竖条按状态变色(正常绿 / 危险黄[血或饱食<半] / 死亡红闪)。
5. **顶栏**:状态徽章(运行正常 / N 警告 / N 故障,按告警聚合)+ 系统运行时长负载 + 时钟 + **「报警历史」按钮**(弹窗看 alerts.log,按级别配色)。
6. **首页不展示日志细节**(日志在详情页)。

### 6.2 详情页(点 MC 卡进入)
- 指标卡 + **实时日志终端**(MC 读 `latest.log`,自动过滤 RCON 连接 spam,按级别高亮)+ **安全检测区**(放置速率排行 + Grim 违规 + 持久化报警日志)+ 在线玩家明细表。

### 6.3 数据来源与关键实现
- **Crafty/进程**:服务状态、java 进程 RSS/uptime。
- **RCON(持久连接,127.0.0.1:25575)**:在线名单、TPS/MSPT(`tps`/`mspt`)、每玩家 血量/饱食/坐标/维度/经验/模式/**护甲**。
  - ⚠️ 1.21 玩家穿戴护甲在 **`equipment.{head,chest,legs,feet}`** 字段,**不在 `Inventory` 100-103 槽**(老写法读不到)。
  - ⚠️ RCON 用**持久连接**(开一次反复用),否则每秒新建连接会把 `latest.log` 刷爆(`Thread RCON Client started/shutting down`)。
- **Server List Ping**:在线人数兜底、MOTD。
- **PCP(性能图)**:`pmrep -a <归档> -o csv` 读 `/var/log/pcp/pmlogger/*/*.index`。CPU% 由 `kernel.all.cpu.idle/(ncpu*1000)` 算,内存% 由 `mem.util.available/physmem`。
  - ⚠️ pmlogger 每天约 00:10 轮转新归档 → 只读最新的午夜后只有几分钟数据 → **合并最近两个归档**。
- **CoreProtect DB**:读 `co_block` 算每玩家近 60 秒放置数(排除 `#` 自然源),≥200/分预警、≥500/分严重(疑似打印机/自动搭建)。
- **GrimAC**:解析 `latest.log` 的违规行接入预警。

### 6.4 预警系统
- **阈值规则**:磁盘>80/90%、内存>85/92%、Swap>85%、CPU≥90% 持续 1 分钟、负载>核数×2、服务 down/unhealthy、TPS<18/14。
- **连通性探测**(独立线程每 25s):直连外网 `cloudflare.com`、MC 隧道 TCP `frp-top.com:18650`。
- **预警自动备份**:内存/Swap/CPU/负载/MC 停止等预警触发时,调 Crafty 备份接口存一次档(3 小时冷却,磁盘/连通性告警不触发)。
- **全报警持久化**:所有报警写入 **`~/dashboard/alerts.log`**(触发记一行、解除记一行,去重,带时间戳/级别),顶栏「报警历史」按钮可查。

## 11. 关键坑 / 经验汇总

| 现象 | 根因 | 解法 |
|---|---|---|
| frpc 登录节点 EOF | mihomo 把国内节点流量绕海外 | mihomo 加 frpc/节点 DIRECT + `--natfrp_tls` |
| Crafty 新建服务器转圈 | 容器 DNS 被 mihomo TUN 吞 + 联网检查写死 ntp/google | `--network host` + 入口 wrapper 打补丁 |
| 改 MC 内存不生效 | Crafty 缓存 execution_command | 重启 **Crafty 容器**(非游戏服) |
| latest.log 被刷爆 | 面板每秒新建 RCON 连接 | RCON 改持久连接 + 日志显示过滤 RCON 行 |
| 护甲全显示空 | 1.21 护甲在 `equipment.*` 不在 Inventory 槽 | 改查 `equipment.head/chest/legs/feet.id` |
| 性能图只有几分钟 | pmlogger 午夜轮转新归档 | 合并最近两个归档 |
| 朋友连不上(invalid session) | 启动器挂了离线档,非正版登录 | 切回正版/微软登录 |
| 朋友连不上(timed out) | 他本地网络(校园网)封端口/路由不通 | 换网络(手机流量热点)/ 或换节点 |
| 刷铁机长时间不刷 | 久离线→区块卸载→村民睡眠记录过期 | 上线等村民睡一觉;或 forceload / 区块加载器 |
